1.背景介绍

JSON Web Token（JWT）是一个开放的标准（RFC 7519），它定义了一个紧凑且自包含的方式，用于在各方之间以JSON对象安全地传输信息。这些信息可以通过数字签名进行验证和信任。可以使用秘密即secret（使用HMAC算法）或使用RSA的公钥/私钥对(a public/private key pair using RSA or ECDSA)来对JWT进行签名。

JWT可以用来做什么？

权限控制：是JWT最广泛的使用场景。一旦用户登录以后，之后的每次请求都会携带被授权的JWT，这样可以允许用户访问路由，服务和资源。

如今通过使用JWT，实现单点登录是一个特色，因为它占用小，并且可以跨越不同的域。

信息交互：采用JWT可以安全的信息交互。因为JWT可以被签名。比如采用私钥/公钥对，可以清楚的知道发送人是谁。另外，因为签名

是通过头部和载荷计算出来的，所以可以自己验证计算，来确认内容是否被篡改。

2.知识剖析

JWT请求流程：


JSON Web Token (JWT)的结构：

Header(头部), Payload(载荷), Signature(签名)。

通常jwt看起来是这样的：xxxxx.yyyyy.zzzzz。


JWT头部通常由两部分组成：token的类型和需要使用的哈希算法。

JWT载荷：包含关于用户的声明语句和其他数据。需要注意的时，载荷部分对所有人是可读的。签名只是为了防止篡改。

JWT签名：由编码后的头部、编码后的载荷、私钥和头部指明的算法计算生成。签名用来辨别JWT内容是否被修改。

共有三种声明类型：registered, public, and private claims。

registered claims：是一系列非强制但推荐使用的预定义声明。提供了一些有用的，可互相操作的的声明。包括iss (issuer), exp (expiration time), sub (subject), aud (audience), and others.

public claims：可以被自由定义但要防止冲突。

private claims：被用户创建的声明用来分享信息。当然只有允许这些private claims时，才有用。
